# 第九步

<h3><a id="user-content-step-8-merging-a-pr" class="anchor" href="https://github.com/cubeton/git101/blob/master/TurtorialInfo/Tutorial.md#step-8-merging-a-pr" aria-hidden="true"></a>合并PR</h3>
<p>继续，然后单击绿色的“合并拉取请求”按钮。这会将你的更改合并到主分支中。</p>

<p>当你完成后，我建议你删除你的分支（太多的分支可能会变得混乱），所以也要点击那个灰色的“删除分支”按钮。</p>

<p>你可以通过单击新仓库第一页上的“提交”链接，仔细检查你的提交是否已合并。</p>

<p>你还可以在右侧看到提交的<a href="https://git-scm.com/docs/git-hash-object" target="_blank"><strong>hash码</strong></a>。Hash码是该特定提交的唯一标识符。它对于引用特定提交和撤消更改（使用 <code><strong><a href="http://git-scm.com/docs/git-revert" target="_blank">git revert</a></strong>&nbsp;&lt;hash code number&gt;</code> 命令撤回），这非常有用。</p>
